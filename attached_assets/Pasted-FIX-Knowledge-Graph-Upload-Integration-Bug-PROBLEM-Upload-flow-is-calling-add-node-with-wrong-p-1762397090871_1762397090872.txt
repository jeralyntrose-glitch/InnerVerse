FIX: Knowledge Graph Upload Integration Bug

PROBLEM:
Upload flow is calling add_node() with wrong parameters, causing:
"KnowledgeGraphManager.add_node() got an unexpected keyword argument 'label'"

SOLUTION:

1. CHECK ACTUAL add_node() SIGNATURE
   Open: /src/services/knowledge_graph_manager.py
   
   Find the add_node() method and check its actual signature.
   It might be one of these:
   
   Option A: async def add_node(self, node_data: dict)
   Option B: async def add_node(self, label: str, type: str, ...)
   Option C: async def add_node(self, **kwargs)

2. FIX THE UPLOAD INTEGRATION
   Open the upload handler file (where we added knowledge graph integration)
   
   Find this section:
```python
   for concept in extracted.get('concepts', []):
       await manager.add_node({

           'source_documents': [document_id]
       })
```
   
   Update it to match the ACTUAL add_node() signature.

```python

       'type': concept['type'],
       'category': concept['category'],

```
